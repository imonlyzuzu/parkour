# Bunnyhop (perfect-timing jump chain) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Jumping again within a tight window of touchdown ("perfect hop") skips the jump-charge cost and adds a flat, capped speed bonus to current momentum; jumping outside that window stays an ordinary full-cost jump.

**Architecture:** A single timing rule — "perfect" means the jump fires within `Config.BhopWindow` seconds of `data.lastLandTime` — covers both a jump buffered just before touchdown (fires instantly on landing in `Airborne.lua`, always perfect by construction) and a jump tapped shortly after landing while already in `Walking`/`Sprinting`. A new tiny `Bhop.luau` helper module centralizes the timing check and the jump+boost execution so `Airborne`, `Walking`, and `Sprinting` don't duplicate it. A new `Stamina:tryConsumePerfectHop()` drains stamina like a normal jump but skips the charge decrement.

**Tech Stack:** Luau, Roblox Studio (verify via MCP console — no headless test runner).

## Global Constraints

- No headless Luau test runner — verification is `mcp__Roblox_Studio__execute_luau` probes against pure logic plus `mcp__Roblox_Studio__get_console_output` after syncing with Rojo. Final feel/playtesting is done by the user in Studio directly, never via MCP.
- `BhopWindow` must be shorter than `Config.JumpBuffer` (0.15) — bhop timing is meant to feel tight, not forgiving.
- `BunnyhopSpeedBonus` is its own Config key, separate from the existing `BhopSpeedBonus` (used by `Sliding.lua`'s crouch-held chain) — do not reuse or rename that key.
- The speed bonus is capped at the existing `Config.TrickMaxSpeed` and uses the existing `data.isTrickSpeed` flag — do not add new cap/decay logic; `FSM._applyLimits` already handles decay back to `MaxGroundSpeed`.
- Do not modify `Landing.luau` (hard-fall roll) or `Sliding.luau` (crouch-chain bhop) — both are out of scope.
- Do not change `Config.JumpSpeed`, `JumpBuffer`, `CoyoteTime`, `StaminaJumpCost`, or `JumpMaxCharges`.

- [ ] **Step 1: Create Bhop.luau**

```lua
-- Bhop.luau
-- Shared helper for the perfect-timing bunnyhop: jumping again within
-- Config.BhopWindow of touchdown skips the jump-charge cost and adds a
-- flat, capped speed bonus to current momentum. Used by Airborne (the
-- instant pre-landing-buffer rebound) and Walking/Sprinting (a tap shortly
-- after landing), so the timing check and boost math live in one place.

local Config = require(script.Parent.Config)

local Bhop = {}

-- True when "now" is still within the perfect-hop window measured from the
-- last touchdown (data.lastLandTime, set by Airborne on landing).
function Bhop.isPerfectWindow(data): boolean
	return os.clock() - (data.lastLandTime or -math.huge) <= Config.BhopWindow
end

-- Fires the jump and adds the flat bunnyhop speed bonus to current planar
-- momentum, capped at TrickMaxSpeed via the shared isTrickSpeed mechanism
-- (FSM._applyLimits already decays it back to MaxGroundSpeed on its own).
-- No-ops the boost (but still jumps) when starting from near-zero speed,
-- since there is no momentum direction to boost along.
function Bhop.perform(world, data)
	world.doJump(Config.JumpSpeed)
	local planar = world.getPlanarVelocity()
	if planar.Magnitude > 1e-3 then
		local boosted = math.min(planar.Magnitude + Config.BunnyhopSpeedBonus, Config.TrickMaxSpeed)
		world.setPlanarVelocity(planar.Unit * boosted)
		data.isTrickSpeed = true
	end
end

return Bhop
```

	local hrp = world.hrp

	-- Touchdown (guard against the rising phase right after a jump).
	if data.isGrounded and data.velocity.Y <= 0.5 then
		-- Stamp the touchdown time first: every branch below (including the
		-- rebound jump a few lines down) is "at touchdown" for bunnyhop
		-- timing purposes, and Walking/Sprinting read this on a later tick
		-- if the player instead taps Jump shortly after landing.
		data.lastLandTime = os.clock()

		-- Crouch held through the jump -> chain straight back into a slide
		-- (b-hop), skipping the roll. Checked before the buffered-Crouch roll
